# Changelog

## [0.4.1] - 2026-06-16

### Fixed
- **上櫃（TPEx）標的抓不到報價**：報價查詢原本對所有台幣標的一律加 `.TW`（上市）後綴，導致在櫃買中心掛牌的債券 ETF（如 00679B／00687B／00772B／00773B 等）抓不到 Yahoo 報價。改為台幣標的先試 `.TW`、抓不到再退而試 `.TWO`（上櫃），並支援使用者自行輸入含後綴的符號（如 `00679B.TWO`）。上市股票（如 0050）行為不變。

## [0.4.0] - 2026-06-15

### Added
- **資產桶完全可自訂**：新增「資產桶設定」對話框，可自行新增／刪除／改名／改色／調整顯示順序。預設仍是五桶，但用不到的桶可以直接刪掉——只用「核心＋防禦」的人，總覽圓餅、持倉明細、走勢圖、再平衡都不會再出現空桶。
  - 收納現金帳戶的「現金桶」標示清楚且不可刪除（確保現金永遠有歸屬）。
  - 仍有持倉的桶不可刪除，會提示先移走持倉，避免產生孤兒資料。
- **持倉可改桶**：持倉明細每列新增桶色點，點一下即可把標的移到其他桶。

### Changed
- 桶從寫死的型別改為可儲存的資料（單一真相來源 `DEFAULT_CATEGORIES`）；總覽、持倉、走勢圖、再平衡、新增交易選單全部改吃使用者自訂的桶。
- 舊存檔開啟時自動補上預設五桶（`loadState` 遷移），既有持倉與快照完全相容、畫面不變。

### Fixed
- **孤兒持倉防呆**：`loadState` 載入時，若持倉的桶 id 不在桶清單裡（只可能來自外部竄改／匯入／sync schema 漂移），自動歸到現金桶，避免市值從總覽默默消失（圓餅加總≠總資產）。正常操作不會觸發。
- **總覽不再出現空桶**：「資產分布」頁的圓餅、長條、卡片改為只顯示「有市值或有設目標」的桶，與持倉桶視圖一致；剛新增還沒放東西的桶不會再跳出「0 萬 / 0.0%」空卡。
- **匯率為 0 時再平衡缺口算出 Infinity**：`delta_usd` 加上 `fx > 0` 防護，避免極端輸入污染美金缺口顯示。
- **跨平台 icon 編譯問題**：原本 `icons/` 只有 `icon.ico`，Linux／macOS 的 bundler 找不到 PNG／icns 導致編譯失敗。改為產生完整跨平台 icon 組合（`32x32.png`／`128x128.png`／`128x128@2x.png`／`icon.icns`／`icon.ico`）並更新 `tauri.conf.json`，三平台皆可編譯。

### Tests
- TypeScript 46：桶遷移（舊存檔無 categories → 自動補預設五桶、持倉不消失；孤兒持倉自動歸現金桶）、桶 CRUD 全套（新增/改名/改色/排序、刪除守門「現金桶不可刪、有持倉不可刪」、setHoldingCategory、改名後 summaries 反映），以及**預設範例的非常規操作穩健性**（全部持倉移到單一桶/現金桶、新增空桶、清空後刪桶、匯率=0、超賣、超額提領、投入新資金分配上限、空組合，皆驗證無 NaN/Infinity、總額不變、各桶加總＝總資產）。`tsc -b`、`vite build`、Rust release 皆綠。

## [0.3.1] - 2026-06-13

### Added
- **再平衡偏離警示門檻可自訂**：原本固定 5%，現在可在「再平衡分析」分頁自行設定（0.5–50%）。首頁的再平衡警示橫幅即時依新門檻重算、文字同步，設定隨退休設定一起存檔。不同人風險偏好不同，門檻自己定。

### Changed
- **總資產卡合併為「台幣(美金)」**：原本分「總資產 (TWD)」「總資產 (USD)」兩張卡，合併為一張——主數字為台幣總額（括號附折合美元），副標另列「台幣資產 / 美元資產」，各以計價幣別的**原值加總**。釐清「總資產(USD)」其實是全資產折算美元、而非美元部位總和的常見誤解。
- **介面色調收斂為品牌橘（#E87930）**：走勢圖區間鈕、交易對話框的選取態改用品牌橘，並調整深色底下虛線按鈕文字對比，與整體暗色主題一致。

### Fixed
- **切換分頁時上方 KPI 卡水平抖動**：不同分頁內容高度不一，導致垂直捲軸出現／消失、頁面寬度跳動，連帶上排卡片位移。改以 `scrollbar-gutter: stable` 永遠預留捲軸寬度，分頁切換版面不再晃。

### Tests
- TypeScript 22（新增 `assetsByCurrency` 幣別拆分與對帳不變量 3 案）；Rust 33，皆綠。

## [0.3.0] - 2026-06-10

### Fixed
- **帳務管家 ↔ 看板雙向同步多項修正**：
  - 內部轉帳（兩端帳戶都對應看板現金帳戶，如富邦 → 元大）合併為**單筆 transfer**，不再各自產生 cash_in / cash_out 兩筆而導致各帳戶餘額對不上
  - 金額為 0 的交易（房貸寬限期本金等）不再同步成雜訊
  - 轉帳的刪除偵測：刪除轉帳任一側都能正確找到並反向還原餘額
  - 同步進來的交易一律帶幣別，避免績效（TWR）換算錯誤
- **修正手動編輯共用資料夾 JSON 留下的 UTF-8 BOM 導致 serde 讀檔失敗、同步靜默中斷**

### Added
- **同步健檢工具**：五大不變量驗證器（冪等 / 引用完整 / 無重複計帳 / 刪除傳播 / 無迴圈）＋ 帳戶餘額對帳（帳務管家連結帳戶 vs 看板快照），可對共用資料夾跑出健檢報告：`cargo run --example sync_check -- <資料夾>`
- **完整回歸測試**：看板 Rust 33 + TypeScript 19；同步契約以 golden fixture 鎖定 Rust↔前端欄位，防止漂移

## [0.1.3] - 2026-05-30

### Added
- **退休里程碑改為「計畫 vs 實際」**：年度／月度里程碑不再只有預測值。計畫值錨定在開始追蹤日（第一張快照）成長，對照實際快照算出「領先/落後計畫」的金額
  - 過去與當前期間顯示真實快照數字 + 對計畫的偏移；未來期間僅顯示計畫值
  - 月度改為「近 12 + 未來 12 個月」，標記本月/今年/退休目標；移除無意義的「距最終目標」欄
- **績效分析新增損益金額**：報酬率（累計 / 今年 YTD / 近一年）旁同時顯示實際損益金額，定義為「期末資產 − 期初資產 − 當期淨存入」，排除本金存入 / 提出，純粹反映投資績效帶來的資金變化
  - 各年度、各月度明細表新增「損益金額」欄
  - 長條圖 tooltip 同時顯示百分比與金額
  - 金額遵守隱藏金額模式
- **帳務管家雙向 Sync**：設定共用根目錄後，資產看板與帳務管家可互相同步資料
  - 看板 → 帳務管家：最新快照的證券市值自動更新帳務管家的 securities 帳戶
  - 帳務管家 → 看板：帳務管家的現金收支自動寫入 `transactions.json`，刪除亦同步反向還原
  - `sync.json` 防止重複匯入；轉帳類交易自動排除

## [0.1.1] - 2026-05-21

### Fixed
- **資料庫膨脹問題**：交易記錄（`transactions`）改存至獨立的 `transactions.json`，不再於每個每日快照重複寫入。舊版每新增一天的快照就會複製一份完整交易歷史，日積月累導致硬碟用量線性成長。

### Migration
升級後首次儲存快照，app 會自動將交易記錄從最新日期檔案抽出，寫入 `transactions.json`，無須手動操作。
